# 模型配置调整指南

## 当前配置

配置文件：`config_yuanbao.yaml`

### 关键参数

```yaml
model:
  path: "yolov8n.pt"           # 模型文件路径
  confidence: 0.6              # 置信度阈值 ⭐ 重要
  input_size: 640              # 输入图像尺寸
  iou_threshold: 0.45          # NMS IoU阈值
  device: "cpu"                # 推理设备
```

## 参数说明

### 1. confidence（置信度阈值）⭐

**作用**: 控制检测的严格程度

- **提高** (0.7-0.8): 只检测非常确定的对象，减少误报
- **降低** (0.3-0.5): 检测更多对象，但可能增加误报

**建议**:
- 如果检测不到对象 → 降低到 0.4-0.5
- 如果误报太多 → 提高到 0.7-0.8
- 默认值: 0.6（平衡点）

### 2. input_size（输入图像尺寸）

**作用**: 影响检测精度和速度

可选值: 320, 416, 512, 640, 768, 1024

- **增大** (768-1024): 检测更精确，但速度更慢
- **减小** (320-416): 速度更快，但可能漏检小对象

**建议**:
- 一般使用: 640（推荐）
- 需要更快: 416
- 需要更准: 768

### 3. iou_threshold（NMS IoU阈值）

**作用**: 控制重复检测框的过滤

- **提高** (0.5-0.7): 允许更多重叠的检测框
- **降低** (0.3-0.4): 更严格地过滤重叠框

**建议**: 保持 0.45（默认值通常很好）

## 常见场景配置

### 场景1: 检测不到对象

```yaml
model:
  confidence: 0.4              # 降低置信度
  input_size: 768              # 增大输入尺寸
```

### 场景2: 误报太多

```yaml
model:
  confidence: 0.75             # 提高置信度
  iou_threshold: 0.5           # 稍微提高IoU
```

### 场景3: 需要更快的速度

```yaml
model:
  input_size: 416              # 减小输入尺寸
  confidence: 0.5              # 稍微降低置信度
```

### 场景4: 需要更高的精度

```yaml
model:
  input_size: 768              # 增大输入尺寸
  confidence: 0.65             # 稍微提高置信度
```

## 测试不同配置

修改 `config_yuanbao.yaml` 后，重新启动：

```bash
python test_yuanbao.py
```

## 查看检测结果

1. 点击 GUI 中的 "🔍 检测" 按钮
2. 查看右侧 "检测结果" 区域
3. 查看运行日志了解检测详情

## 训练专用模型

如果需要识别元宝中的特定对象：

1. **收集数据**: 截取元宝游戏画面
2. **标注数据**: 使用 LabelImg 等工具标注对象
3. **训练模型**: 使用 YOLOv8 训练专用模型
4. **替换模型**: 将训练好的模型文件放到项目目录
5. **更新配置**: 修改 `model.path` 指向新模型

## 诊断工具

如果画面显示有问题，运行：

```bash
python diagnose_capture.py
```

这会生成截图文件，帮助诊断问题。
