# 工单详细设计 Skill

## 功能概述

本skill能够理解需求文档，并按照需求文档的要求严格撰写出详细设计文档。它提供专业的流程图、用例图、数据流图、跨职能流程图、时序图等专业图表，并结合Flowable工作流引擎和自定义表单功能，形成符合用户场景的二次开发功能。

## 主要功能

### 1. 需求分析与理解
- 深入分析需求文档的业务逻辑和功能要求
- 识别核心业务流程和关键业务规则
- 提取功能需求和非功能需求
- 明确系统边界和接口定义

### 2. 详细设计文档生成
根据需求文档生成包含以下内容的详细设计文档：

#### 2.1 系统架构设计
- 整体架构图
- 模块划分与职责
- 技术选型说明
- 部署架构设计

#### 2.2 业务流程设计
提供至少一种专业图表（根据需求选择）：
- **流程图**：展示业务流程的步骤和决策点
- **用例图**：描述系统功能和用户交互
- **数据流图**：展示数据在系统中的流动
- **跨职能流程图**：展示不同角色间的协作
- **时序图**：展示组件间的交互顺序

#### 2.3 Flowable工作流集成
- 工作流程设计
- 流程节点定义
- 流程变量配置
- 任务分配策略
- 流程监控与优化

#### 2.4 自定义表单设计
- 表单字段定义
- 表单验证规则
- 表单布局设计
- 表单与流程集成
- 表单数据绑定

### 3. 影响分析Checklist
提供完整的检查清单，帮助开发者检查新功能对现有系统的影响：

#### 3.1 事件影响
- [ ] 是否需要新增/修改事件定义
- [ ] 是否会影响现有事件监听器
- [ ] 事件数据结构是否需要变更
- [ ] 异步事件处理是否需要调整

#### 3.2 级联影响
- [ ] 是否会影响关联模块的数据
- [ ] 是否需要更新级联删除/更新规则
- [ ] 是否会影响数据一致性
- [ ] 是否需要处理级联异常

#### 3.3 基础数据影响
- [ ] 是否需要新增基础数据
- [ ] 是否需要修改现有基础数据结构
- [ ] 是否会影响数据字典
- [ ] 是否需要数据迁移脚本

#### 3.4 报告中心影响
- [ ] 是否需要新增报表
- [ ] 是否会影响现有报表数据
- [ ] 是否需要修改报表模板
- [ ] 是否需要调整报表权限

#### 3.5 接口影响
- [ ] 是否需要新增API接口
- [ ] 是否会影响现有API
- [ ] 是否需要修改接口版本
- [ ] 是否需要更新接口文档

#### 3.6 性能影响
- [ ] 是否会影响系统性能
- [ ] 是否需要优化数据库查询
- [ ] 是否需要添加缓存机制
- [ ] 是否需要调整并发处理

### 4. API定义生成
生成完善的API接口定义：

#### 4.1 接口基本信息
- 接口名称
- 接口描述
- 请求方式（GET/POST/PUT/DELETE）
- 接口路径
- 认证方式

#### 4.2 请求参数
- 参数名称
- 参数类型
- 是否必填
- 参数说明
- 示例值
- 验证规则

#### 4.3 响应参数
- 参数名称
- 参数类型
- 参数说明
- 示例值
- 错误码定义

#### 4.4 接口示例
- 请求示例
- 响应示例
- 错误响应示例

### 5. 数据库设计
如果需要数据库改造，提供完整的数据库设计章节：

#### 5.1 数据库设计原则
- 规范化设计
- 性能优化考虑
- 数据完整性约束
- 索引设计原则

#### 5.2 表结构设计
- 表名定义
- 字段定义
- 字段类型
- 约束条件
- 索引设计
- 外键关系

#### 5.3 数据字典
- 字段含义说明
- 数据类型说明
- 取值范围说明
- 业务规则说明

#### 5.4 数据迁移
- 迁移脚本
- 数据转换逻辑
- 回滚方案
- 迁移验证

### 6. 文档格式要求
最终生成的文档格式为Word文档（.docx），包含以下章节：

1. **文档信息**
   - 文档版本
   - 编写日期
   - 编写人员
   - 审核人员

2. **需求概述**
   - 需求背景
   - 需求目标
   - 需求范围

3. **系统设计**
   - 系统架构
   - 模块设计
   - 技术架构

4. **业务流程设计**
   - 流程图
   - 流程说明
   - 业务规则

5. **功能设计**
   - 功能模块
   - 功能描述
   - 功能逻辑

6. **接口设计**
   - API定义
   - 接口说明
   - 接口示例

7. **数据库设计**（如需要）
   - 表结构设计
   - 数据字典
   - 迁移方案

8. **影响分析**
   - 影响评估
   - 风险分析
   - 应对措施

9. **附录**
   - 参考文档
   - 术语表
   - 变更记录

## 使用指南

### 输入要求
用户提供需求文档，可以是以下格式：
- Word文档
- PDF文档
- 文本文件
- 在线文档链接

### 输出要求
生成完整的详细设计Word文档，包含：
- 专业的流程图和架构图
- 完整的API接口定义
- 详细的影响分析checklist
- 必要的数据库设计
- 清晰的技术方案

### 使用流程
1. 用户上传或提供需求文档
2. Skill分析需求文档内容
3. 识别关键业务需求和技术要求
4. 生成详细设计文档框架
5. 填充具体设计内容
6. 生成专业图表
7. 完成影响分析checklist
8. 生成API接口定义
9. 如需要，提供数据库设计
10. 导出为Word文档

## 技术特点

### Flowable集成
- 支持BPMN 2.0标准
- 提供可视化流程设计
- 支持流程版本管理
- 提供流程监控和分析

### 自定义表单
- 支持动态表单生成
- 提供表单验证机制
- 支持表单与流程绑定
- 提供表单模板管理

### 图表生成
- 使用专业绘图工具
- 支持多种图表类型
- 符合UML标准
- 提供图表导出功能

## 质量保证

- 文档完整性检查
- 技术方案可行性验证
- 影响分析全面性评估
- API接口规范性检查
- 数据库设计合理性审核

## 注意事项

1. 确保需求文档的完整性和准确性
2. 与相关业务方确认需求理解
3. 定期与开发团队沟通设计细节
4. 及时更新文档以反映需求变更
5. 保持文档与实际实现的一致性
